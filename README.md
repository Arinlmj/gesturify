# GESTURIFY

GESTURIFY 是一个基于真实摄像头手势识别的粒子特效控制台。页面通过 MediaPipe Hand Landmarker 实时追踪手部关键点，用「捏合 / 抓取」和「释放」手势控制右侧 Three.js 粒子场，实现由远及近聚合、抓取拖动旋转、释放后球形绽放的交互效果。

## 功能特性

- 真实摄像头输入：使用浏览器 `getUserMedia` 获取摄像头画面，不使用模拟手势数据。
- 手势识别：通过 MediaPipe 识别手部骨架、关键点、拇指与食指距离。
- 稳定抓取状态：释放需要连续确认，短暂丢帧会保持抓取，减少旋转或遮挡导致的误释放。
- 粒子场聚合：抓取后远处、中层、近处粒子分层向手势点收束，形成由远及近的聚合感。
- 拖动旋转：保持抓取后左右或斜向移动手势点，可控制粒子场旋转速度。
- 统一释放效果：释放时从聚合粒子核向外球形绽放，避免条状外放。
- 可调参数：支持能量、粒子密度、旋转灵敏度与抓取样式切换。

## 本地运行

```bash
npm install
npm run dev
```

启动后访问终端输出的本地地址，默认类似：

```text
http://127.0.0.1:5173/
```

如果端口被占用，Vite 会自动切换到其他端口，例如 `http://127.0.0.1:5176/`。

## 使用方式

1. 点击「启动摄像头」。
2. 在浏览器权限弹窗中允许摄像头访问。
3. 把手放入摄像头画面，等待状态进入「追踪中」。
4. 拇指和食指捏合进入「抓取」，粒子会由远及近聚合到手势点。
5. 保持捏合并左右或斜向拖动手势点，控制粒子场旋转。
6. 完全松开拇指和食指，粒子会从聚合核心向外球形绽放。

## Docker / 极空间部署

项目已提供 `Dockerfile`、`nginx.conf` 和 `docker-compose.yml`，适合在极空间 Docker 中部署为静态站点。

在支持 Docker Compose 的环境中运行：

```bash
docker compose up -d --build
```

默认映射端口：

```text
宿主机 18080 -> 容器 80
```

访问地址示例：

```text
http://极空间-ip:18080/
```

注意：浏览器摄像头能力要求安全上下文。通过局域网 IP 的普通 HTTP 访问时，部分浏览器会禁止摄像头；建议在极空间上绑定 HTTPS 域名或通过反向代理启用 HTTPS。

## 摄像头与模型说明

摄像头能力依赖浏览器安全上下文。本地 `127.0.0.1` 可以直接调用摄像头；线上部署时需要 HTTPS。

手势模型使用 MediaPipe 官方模型地址加载：

```text
https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/1/hand_landmarker.task
```

如果模型加载失败，请检查网络是否可以访问 MediaPipe WASM 与模型文件。

## 常见问题

- 摄像头权限被拒绝：检查浏览器地址栏权限、系统隐私设置，或重新授权摄像头。
- 摄像头被占用：关闭其他正在使用摄像头的软件后重试。
- 抓取时误释放：尽量保持拇指和食指稳定捏合；当前实现已加入连续释放确认与短暂丢帧保持。
- 旋转不明显：提高「旋转灵敏度」，并在抓取状态下左右或斜向拖动手势点。

更多操作细节见 [USAGE.md](./USAGE.md)。
